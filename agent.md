# WhereCode Agent DNA

## Non-negotiable order (every task)
1. Update plan first (`PLAN.md` + task board when needed).
2. Implement changes.
3. Run checks.
4. Update docs.

## Ownership rule
- AI executes end-to-end delivery.
- User provides goals/suggestions only.

## Task logging rule
- Start: add `started (doing)`.
- Finish: mark `completed (done)` directly.
- Do not add completion narrative.
- If interrupted/blocked, add one short blocker note.

## Doc style rule
- Keep docs short and operational.
- No filler sections.
- No "conclusion" for completed tasks.
- Keep only scope, commands, and state.

## Execution rule
- Prefer full delivery over minimal diff.
- Follow repo conventions.
- If checks cannot run, log blocker + manual verify command.
