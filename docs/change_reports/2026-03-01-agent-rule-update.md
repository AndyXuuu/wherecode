# 变更总结：Agent 执行纪律升级

日期：2026-03-01

## 1. 变更目标

将仓库内 Agent 执行规则升级为强制三阶段流程：

1. 先更新计划
2. 再实施改动
3. 最后写总结文档

## 2. 计划更新

- 更新 `PLAN.md`：
  - 在核心原则新增“三阶段执行纪律”。
  - 新增“执行纪律（强制）”章节，定义闭环标准与落地路径。

## 3. 实施改动

- 更新 `agent.md`：
  - 调整标准工作流，加入 `Plan First` 与 `Summary Last` 两个强制步骤。
  - 在执行约束中增加“固定顺序”规则。
- 更新 `docs/README.md`：
  - 增加 `change_reports/` 目录说明。
- 新增 `docs/change_reports/README.md`：
  - 约定变更总结文档命名和模板。

## 4. 验证结果

- 文档结构检查通过：规则入口、流程说明、总结目录已建立。
- 本次改动已按新规则闭环：先计划、后改动、再总结。

## 5. 风险与后续事项

- 风险：当前为流程与规范落地，尚未在代码层硬性校验（如 CI 检查“是否有对应总结文档”）。
- 后续建议：增加自动化校验脚本，阻止未更新计划或未新增总结文档的提交。
